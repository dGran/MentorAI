Bits y bytes: cómo cuenta de verdad un ordenador

Un número, un texto, una imagen, este tutorial: para la máquina todo es lo mismo, una tira de ceros y unos. Entender ese nivel no es trivia académica; es lo que hace que de pronto cobren sentido los permisos en octal, las máscaras de flags, los colores en hex y por qué a veces los datos "no caben".

Por debajo de todos los tipos de tu lenguaje —enteros, cadenas, booleanos— hay una única cosa: bits agrupados en bytes. Casi cada "rareza" que te encuentras (permisos 0644, un color #ff8800, un & que no es &&) es transparente en cuanto ves la tira de bits que hay debajo.

Del bit al byte #

Un bit es la unidad mínima de información: vale 0 o 1, nada más. Con un solo bit distingues dos estados (encendido/apagado, sí/no). El truco está en agruparlos: cada bit que añades duplica las combinaciones posibles. Con 2 bits hay 4 combinaciones; con 3, ocho; con n bits, 2ⁿ.

1 Bit1 dígito: 0 o 1 (2 estados)
4 Nibble4 bits: 16 valores (un dígito hex)
8 Byte8 bits: 256 valores (0–255)

La unidad de trabajo es el byte: 8 bits, 2⁸ = 256 valores posibles.

El byte (8 bits) es la unidad direccionable de memoria: cuando un lenguaje dice que un char ocupa 1 byte o que un entero ocupa 4 u 8, habla de cuántos de estos grupos de 8 bits reserva. Un byte cubre del 0 al 255: por eso ese número aparece por todas partes (un canal de color RGB, un octeto de una IP 192.168.0.1…).

¿Por qué 8 y no 10?

Porque el hardware no tiene diez dedos: un transistor está conducido o cortado, hay tensión o no la hay. Dos estados, no diez. Toda la electrónica digital se construye sobre esa decisión física, y por eso contamos en base 2, no en base 10.

Contar en binario #

El sistema decimal es posicional: en 237 el 2 vale 200 porque está en la posición de las centenas (10²). El binario funciona igual, pero cada posición vale el doble que la anterior en vez de diez veces más: son potencias de 2.

Posición2⁷2⁶2⁵2⁴2³2²2¹2⁰
Vale1286432168421
0110 010101100101

Sumas las posiciones que tienen un 1: 64 + 32 + 4 + 1 = 101. Así de mecánico es leer binario. Para ir y volver no hace falta hacerlo a mano: todo lenguaje trae conversiones, y verlas escritas ayuda a fijar la idea.

php · binario ⇄ decimal 
<?php

declare(strict_types=1);

$value = 101;

echo decbin($value);            // "1100101"
echo bindec('1100101');         // 101

printf('%08b', $value);         // "01100101" (8 bits, con ceros a la izquierda)

$literal = 0b01100101;          // binario directo en el código
var_dump($literal === 101);     // bool(true)

Hexadecimal: binario para humanos #

El binario es fiel a la máquina pero ilegible para nosotros: 11111111 cuesta de leer. El hexadecimal (base 16) es el atajo: como 16 es 2⁴, cada dígito hex equivale exactamente a 4 bits (un nibble). Un byte son siempre dos dígitos hex, sin ambigüedad. Por eso lo verás en colores, hashes, direcciones de memoria y volcados de bytes.

Leer un color de un vistazo

#FF8800 son tres bytes: FF = 255 de rojo, 88 = 136 de verde, 00 = 0 de azul. Cada par hex es un byte, cada byte un canal de 0 a 255. No hay magia: es el mismo conteo de siempre, escrito compacto.

php · hexadecimal y bytes 
<?php

declare(strict_types=1);

echo dechex(255);               // "ff"
echo hexdec('ff');              // 255

$red = hexdec('FF');            // 255
$green = hexdec('88');          // 136

$literal = 0xFF;                // hex directo en el código
var_dump($literal === 255);     // bool(true)

Operaciones bit a bit #

Los operadores bitwise trabajan sobre cada bit por separado, sin acarreo. Son la base de las máscaras de la siguiente sección, así que conviene tenerlos claros. Hay cinco que importan:

OperadorHaceEjemplo (4 bits)
& AND 1 solo si ambos bits son 1 1100 & 1010 = 1000
| OR 1 si alguno de los dos es 1 1100 | 1010 = 1110
^ XOR 1 si los dos bits difieren 1100 ^ 1010 = 0110
~ NOT Invierte todos los bits ~1100 = …0011
<< / >> shift Desplaza los bits a izq./der. 0001 << 2 = 0100

Un detalle útil: desplazar a la izquierda una posición (<< 1) equivale a multiplicar por 2, y a la derecha (>> 1) a dividir entre 2 descartando el resto. No es casualidad: en base 2 mover la coma es multiplicar o dividir por la base, igual que añadir un cero en decimal multiplica por 10.

php · los cinco operadores 
<?php

declare(strict_types=1);

printf('%04b', 12 & 10);        // "1000"  (AND)
printf('%04b', 12 | 10);        // "1110"  (OR)
printf('%04b', 12 ^ 10);        // "0110"  (XOR)

echo 1 << 4;                    // 16   (1 desplazado 4 posiciones)
echo 20 >> 2;                   // 5    (20 / 4)

Flags y máscaras: el caso real #

Aquí es donde todo lo anterior se vuelve práctico. Una máscara de bits guarda muchos booleanos independientes en un solo entero: cada bit es un flag. Es el patrón de los permisos de Unix (rwx → 3 bits), de las opciones de muchas funciones de PHP (JSON_PRETTY_PRINT | JSON_UNESCAPED_UNICODE) y de cualquier conjunto de opciones que quepa en un número.

El idioma es siempre el mismo: cada flag es una potencia de 2 (un único bit a 1), se combinan con OR, se comprueban con AND y se quitan con AND del complemento (& ~flag).

php · permisos como máscara de bits 
<?php

declare(strict_types=1);

const READ = 1 << 0;            // 0b001 = 1
const WRITE = 1 << 1;           // 0b010 = 2
const DELETE = 1 << 2;          // 0b100 = 4

$permissions = READ | WRITE;    // 0b011 = 3: lectura y escritura

$canWrite = ($permissions & WRITE) !== 0;     // true
$canDelete = ($permissions & DELETE) !== 0;   // false

$permissions = $permissions & ~WRITE;         // quita WRITE -> 0b001
$permissions = $permissions | DELETE;          // añade DELETE -> 0b101

Una máscara entera frente a varias columnas booleanas

Guardar 8 flags en un solo INT es compacto y rápido de combinar, pero ilegible en la base de datos y difícil de indexar o consultar (WHERE permisos & 2 no usa índice). Para pocas opciones muy consultadas, columnas o un enum suelen ganar; la máscara brilla cuando son muchas y se manejan en bloque.

Errores comunes #

ErrorPor qué duele
Confundir & con && & opera bit a bit; && es lógico (AND booleano). 6 & 1 es 0; 6 && 1 es true.
Flags que no son potencias de 2 Si dos flags comparten un bit, se pisan: comprobar uno da positivos falsos. Cada flag = un único bit (1 << n).
Olvidar el !== 0 al comprobar $x & FLAG devuelve el valor del bit (p. ej. 4), no true. Funciona por casualidad hasta que comparas o sumas.
Asumir que un entero es infinito Cabe en un número fijo de bits; al desbordar, se rompe o cambia de signo. Es justo el siguiente tutorial.

Resumen #

Lo que te llevas

  • Un bit es 0 o 1; ocho forman un byte (256 valores). Cada bit que añades duplica las combinaciones (2ⁿ).
  • El binario es posicional con potencias de 2; el hex es su atajo legible: un dígito hex = 4 bits, un byte = dos dígitos.
  • Los operadores bit a bit (AND, OR, XOR, NOT, shifts) trabajan cada bit por su cuenta; << 1 multiplica por 2.
  • Las máscaras guardan muchos flags en un entero: combinar con |, comprobar con &, quitar con & ~flag.
  • & no es &&, los flags deben ser potencias de 2, y un entero no es infinito: cabe en sus bits.
