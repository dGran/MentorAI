Nombres con intención

Si solo pudieras mejorar una cosa de tu código, mejora los nombres. Un buen nombre hace innecesario el comentario, convierte una línea oscura en una frase legible y le ahorra al lector el trabajo de adivinar. Es la inversión con más interés de todo el código limpio.

Un nombre debería contestar tres preguntas a la vez: por qué existe esto, qué hace y cómo se usa. Si para entenderlo necesitas un comentario al lado, el nombre ha fallado. La buena noticia: poner buenos nombres no cuesta talento, cuesta atención y un par de reglas.

Revelar la intención #

El nombre debe decir qué representa, no qué tipo tiene ni cómo está implementado. $d no dice nada; $diasDesdeUltimoAcceso lo dice todo. El coste de teclear más letras es ridículo frente al de descifrar la intención cada vez que se lee.

Sin intención

  • $list, $data, $tmp
  • function calc($x, $y)
  • $flag = true

Con intención

  • $pedidosPendientes
  • function precioConIva($base, $tipo)
  • $haExpirado = true
php · el mismo bucle, con y sin intención 
<?php

// Opaco: ¿qué es l? ¿qué es x?
$resultado = [];
foreach ($l as $x) {
    if ($x[3] === 1) {
        $resultado[] = $x;
    }
}

// Claro: el nombre cuenta la historia
$usuariosActivos = [];
foreach ($usuarios as $usuario) {
    if ($usuario->estaActivo()) {
        $usuariosActivos[] = $usuario;
    }
}

Evitar la desinformación #

Peor que un nombre vago es uno que miente. Llamar $listaUsuarios a algo que no es una lista, o $cuentaActiva a una variable que a veces guarda cuentas inactivas, planta una trampa para el lector. El nombre es un contrato: cuando dice una cosa y el código hace otra, gana la confusión.

Cuidado con los nombres parecidos

$cuentaOrigen y $cuentaOrigenCuenta a tres líneas de distancia son una bomba de relojería: el ojo no los distingue y se mezclan. Nombres que se diferencian de un vistazo evitan errores tontos y caros.

Pronunciables y buscables #

Si no puedes pronunciar un nombre, no puedes hablar de él en una revisión sin deletrearlo. $fchaModfccn es impronunciable; $fechaModificacion se dice y se entiende. Y deben ser buscables: un 7 mágico repartido por el código es imposible de encontrar; una constante MAXIMO_INTENTOS aparece con una búsqueda.

php · números mágicos vs constantes buscables 
<?php

// Mágico: ¿qué es 86400? ¿y por qué 3?
if ($antiguedad > 86400 * 3) { /* ... */ }

// Buscable y autoexplicado
private const SEGUNDOS_POR_DIA = 86400;
private const DIAS_DE_GRACIA = 3;

if ($antiguedad > self::SEGUNDOS_POR_DIA * self::DIAS_DE_GRACIA) { /* ... */ }

No abreviar #

Las abreviaturas ahorran teclas y cuestan comprensión. $usr, $btn, $idx, $e obligan al lector a traducir mentalmente. Escribe $usuario, $boton, $indice, $exception. La excepción son las siglas de dominio universalmente conocidas ($url, $id, $iva): esas son el nombre, no una abreviatura.

Un nombre que pide abreviarse delata algo

Si un nombre honesto te sale larguísimo ($gestorDeProcesamientoDeFacturasPendientes), no abrevies: probablemente la clase o el método hacen demasiado. Es la conexión con Object Calisthenics y con la responsabilidad única: el nombre hinchado es síntoma de una responsabilidad hinchada.

Booleanos y convenciones #

Las convenciones de naming dejan que el lector deduzca el tipo y la intención sin pensar. La más útil: los booleanos se leen como una pregunta de sí/no, con prefijo is, has o should.

Tipo de nombre Convención Ejemplo
Booleano is / has / should isActive, hasPermission, shouldRetry
Clase Sustantivo Factura, RepositorioUsuarios
Método con efecto Verbo guardar(), enviar()
Método que consulta Verbo de consulta total(), estaVacio()
php · un booleano bien nombrado se lee como una frase 
<?php

// Ambiguo: ¿active es un estado, una acción, un objeto?
if ($active && $perm) { /* ... */ }

// Se lee solo
if ($usuario->isActive() && $usuario->hasPermission('borrar')) {
    // ...
}

Resumen #

Lo que te llevas

  • Un nombre debe revelar la intención: qué representa, no qué tipo tiene. Si necesita un comentario, ha fallado.
  • Evita los nombres que mienten o se confunden entre sí; usa nombres pronunciables y buscables (constantes, no números mágicos).
  • No abrevies (salvo siglas de dominio); un nombre que pide abreviarse delata una responsabilidad hinchada.
  • Sigue las convenciones: booleanos con is/has/should, clases sustantivo, métodos verbo.
